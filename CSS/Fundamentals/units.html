CSS Unit Differences

This page demonstrates the behavior of different CSS units: `px` (pixels), `%` (percentage), `vh` (viewport height), and `rem` (root em). Resize your browser window to see how each unit responds!

150px Width / 100px Height
(Absolute, fixed size)

Pixels (px): This is an absolute unit. A pixel is a fixed size (usually 1/96th of an inch) and does not scale with the viewport or font size. Elements styled with `px` will maintain their exact dimensions regardless of screen size or user preferences, which can lead to layout issues on different devices.

50% Width / 20% Height
(Relative to parent)

Percentage (%): This is a relative unit, always calculated based on the parent element's size. For width, it's relative to the parent's width; for height, it's relative to the parent's height. If the parent's size changes (e.g., due to resizing the browser), the element's size will change proportionally. Note that height percentages require the parent to have a defined height.

30vw Width / 20vh Height
(Relative to viewport dimensions)

Viewport Height/Width (vh/vw): These are relative units based on the dimensions of the browser's viewport.

  • 1vh equals 1% of the viewport's height.
  • 1vw equals 1% of the viewport's width.
This makes elements scale directly with the browser window size, ideal for full-screen sections or responsive layouts that need to adapt to the visible screen area.

10rem Width / 5rem Height
(Relative to root font-size)

Root EM (rem): This is a relative unit based on the `font-size` of the root HTML element (``). By default, browsers set the root `font-size` to 16px, so 1rem = 16px. If a user changes their browser's default font size or if you explicitly set `font-size` on the `html` element, `rem` values will scale accordingly, making them excellent for accessible and scalable typography and layouts.
